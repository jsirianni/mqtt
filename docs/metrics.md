# Metrics

`mqtt` exposes Prometheus-compatible metrics using OpenTelemetry.

## Endpoint

- Path: `/metrics`
- Default listen address: `:9464`
- Enable/disable: `metrics.enabled`

Example:

```bash
curl http://localhost:9464/metrics
```

## Lifecycle And Availability Signals

These are designed to work with pull scraping (for example, every 60 seconds):

- `mqtt_process_start_time_seconds` (gauge): process start Unix timestamp.
- `mqtt_server_uptime_seconds` (gauge): seconds since process start.
- `mqtt_listener_up` (gauge): `1` when MQTT listener is serving, `0` otherwise.
- `mqtt_listener_errors_total` (counter): listener bind/accept failures.
- `mqtt_build_info` (gauge): always `1` with static labels for build metadata.

Notes:

- Event counters like "start" and "stop" are intentionally not used as primary lifecycle signals.
- Use `mqtt_process_start_time_seconds`, `mqtt_server_uptime_seconds`, and target `up` for robust restart/availability detection.
- Graceful shutdown should be treated as a structured log event, not a guaranteed pull metric event.

## Metric Catalog

### Process/Service

- `mqtt_process_start_time_seconds` (gauge, unit: `s`): process start timestamp for restart detection.
  - Labels: none
- `mqtt_server_uptime_seconds` (gauge, unit: `s`): elapsed process runtime.
  - Labels: none
- `mqtt_listener_up` (gauge): listener readiness state (`1` or `0`).
  - Labels: none
- `mqtt_listener_errors_total` (counter): listener failures in bind/accept path.
  - Labels: none
- `mqtt_build_info` (gauge): constant `1` with build metadata for rollout correlation.
  - Labels: `version`, `revision`

### Connection Health

- `mqtt_connections_accepted_total` (counter): cumulative accepted client sockets.
  - Labels: none
- `mqtt_connections_active` (gauge): current active connections.
  - Labels: none
- `mqtt_connections_closed_total` (counter): connection closes by mode.
  - Labels: `mode` (`graceful`, `ungraceful`, `takeover`)
- `mqtt_connection_duration_seconds` (histogram, unit: `s`): connection lifetime distribution.
  - Labels: none

### Protocol Traffic

- `mqtt_packets_in_total` (counter): inbound MQTT packets by type.
  - Labels: `packet_type`
- `mqtt_packets_out_total` (counter): outbound MQTT packets by type.
  - Labels: `packet_type`
- `mqtt_packet_size_bytes` (histogram, unit: `By`): packet size by direction.
  - Labels: `direction` (`in`, `out`)
- `mqtt_decode_errors_total` (counter): decode failures by stage and reason.
  - Labels: `stage`, `reason`
- `mqtt_encode_errors_total` (counter): encode/write failures by packet type and reason.
  - Labels: `packet_type`, `reason`

### Broker Behavior

- `mqtt_connect_attempts_total` (counter): CONNECT attempts handled.
  - Labels: none
- `mqtt_connect_rejections_total` (counter): rejected CONNECT attempts.
  - Labels: `reason_code` (hex MQTT reason code, for example `0x82`)
- `mqtt_publish_in_total` (counter): inbound publish volume by QoS/retain.
  - Labels: `qos`, `retain`
- `mqtt_publish_forwarded_total` (counter): forwarded deliveries by effective QoS.
  - Labels: `qos`
- `mqtt_publish_matches_total` (counter): total subscription matches generated by routing.
  - Labels: none
- `mqtt_publish_offline_queued_total` (counter): publishes queued due to offline or flow-limited receivers.
  - Labels: none
- `mqtt_subscribe_total` (counter): subscribe requests handled.
  - Labels: none
- `mqtt_unsubscribe_total` (counter): unsubscribe requests handled.
  - Labels: none
- `mqtt_subscribe_rejections_total` (counter): subscribe rejections.
  - Labels: `reason_code` (hex MQTT reason code)

### Backpressure/Delivery

- `mqtt_outbound_queue_drops_total` (counter): dropped/deferred non-blocking sends.
  - Labels: `path`
- `mqtt_inflight_messages` (gauge): aggregate QoS1 in-flight deliveries awaiting PUBACK.
  - Labels: none
- `mqtt_pending_messages` (gauge): aggregate QoS1 pending queue depth.
  - Labels: none
- `mqtt_puback_latency_seconds` (histogram, unit: `s`): QoS1 send-to-PUBACK latency.
  - Labels: none

### Session State

- `mqtt_sessions_active` (gauge): number of tracked sessions.
  - Labels: none
- `mqtt_sessions_persistent` (gauge): number of persistent sessions.
  - Labels: none
- `mqtt_sessions_clean` (gauge): number of clean-session clients.
  - Labels: none

## Cardinality Policy

Allowed labels are bounded enum-like sets:

- `packet_type`, `qos`, `retain`, `mode`, `reason_code`, `direction`, `stage`, `path`, `version`, `revision`

Do not add high-cardinality labels:

- `client_id`
- topic names or filters
- `username`
- `remote_addr`
- arbitrary error strings or message payload content

## Sample PromQL

Connection churn rate:

```promql
rate(mqtt_connections_accepted_total[5m])
```

Ungraceful close ratio:

```promql
rate(mqtt_connections_closed_total{mode="ungraceful"}[5m])
/
rate(mqtt_connections_closed_total[5m])
```

Publish ingest throughput:

```promql
sum by (qos, retain) (rate(mqtt_publish_in_total[5m]))
```

Queue pressure:

```promql
mqtt_inflight_messages + mqtt_pending_messages
```

QoS1 PUBACK p95 latency:

```promql
histogram_quantile(0.95, sum by (le) (rate(mqtt_puback_latency_seconds_bucket[5m])))
```

Restart detection signal:

```promql
changes(mqtt_process_start_time_seconds[1h])
```
